# 관리자 기능 시퀀스 (Mermaid)

WeTube 관리자(/admin) 접근·조회·수정·삭제 흐름을 시퀀스 다이어그램으로 정리합니다.

---

## 1. /admin 접근 시퀀스

사용자가 `/admin`에 접근할 때 인증·권한에 따른 분기입니다.

```mermaid
sequenceDiagram
    participant U as 사용자(브라우저)
    participant F as Flask (admin 라우트)
    participant A as _admin_required
    participant DB as DB

    U->>F: GET /admin
    F->>A: @_admin_required 적용

    alt 비로그인
        A->>A: current_user.is_authenticated == False
        A->>F: redirect(auth.login, next=/admin)
        F->>U: 302 → /auth/login
    else 로그인 + 일반 유저
        A->>A: current_user.is_admin == False
        A->>F: flash("관리자만 접근할 수 있습니다.")
        A->>F: redirect(main.index)
        F->>U: 302 → / (메인)
    else 로그인 + 관리자
        A->>F: f(*args, **kwargs) 통과
        F->>DB: User/Video/Comment count, list(20건)
        DB->>F: stats, users_list, videos_list, comments_list
        F->>U: 200 admin/admin.html (통합 관리 화면)
    end
```

---

## 2. 관리자 대시보드 → 세부 목록 이동

통합 관리 화면에서 "전체 보기"로 사용자/동영상/댓글 목록 페이지로 이동하는 흐름입니다.

```mermaid
sequenceDiagram
    participant U as 관리자
    participant Admin as /admin (admin.html)
    participant Users as /admin/users
    participant Videos as /admin/videos
    participant Comments as /admin/comments

    U->>Admin: GET /admin
    Admin->>U: 통계 + 사용자/동영상/댓글 표(각 20건)

    U->>Users: 클릭 "사용자 전체"
    Users->>U: GET /admin/users (페이지네이션)

    U->>Videos: 클릭 "동영상 전체"
    Videos->>U: GET /admin/videos (페이지네이션)

    U->>Comments: 클릭 "댓글 전체"
    Comments->>U: GET /admin/comments (페이지네이션)
```

## 3. 회원 수정 시퀀스 (관리자)

관리자가 특정 회원의 정보(이메일, 닉네임, 비밀번호, 프로필 이미지, 관리자 여부)를 수정하는 흐름입니다.

```mermaid
sequenceDiagram
    participant U as 관리자
    participant F as admin.user_edit
    participant DB as DB
    participant Auth as auth._save_profile_image

    U->>F: GET /admin/users/<user_id>/edit
    F->>DB: User.query.get_or_404(user_id)
    DB->>F: edit_user
    F->>U: 200 user_edit.html (폼)

    U->>F: POST /admin/users/<user_id>/edit (email, nickname, is_admin, password, profile_image)
    F->>F: 이메일 필수·중복 검사
    F->>F: 비밀번호 변경 시 확인 일치·4자 이상 검사

    alt 프로필 이미지 업로드 있음
        F->>Auth: _save_profile_image(profile_file, user)
        Auth->>F: (Cloudinary 또는 로컬 저장)
    end

    F->>F: 자기 자신이면 is_admin=True 유지, 아니면 폼값 반영
    F->>DB: user.nickname, user.email, user.is_admin, db.session.commit()
    DB->>F: OK
    F->>U: redirect(admin.index) + flash("회원 정보가 수정되었습니다.")
```

## 4. 회원 삭제 시퀀스

관리자가 회원 삭제 버튼을 눌렀을 때의 흐름입니다. 자기 자신은 삭제할 수 없습니다.

```mermaid
sequenceDiagram
    participant U as 관리자
    participant F as admin.user_delete
    participant DB as DB

    U->>F: POST /admin/users/<user_id>/delete (csrf_token)
    F->>F: user_id == current_user.id ?

    alt 자기 자신
        F->>U: flash("자기 자신은 삭제할 수 없습니다."), redirect(admin.index)
    else 다른 회원
        F->>DB: User.query.get_or_404(user_id)
        F->>DB: db.session.delete(user), db.session.commit()
        DB->>F: OK
        F->>U: flash("회원이 삭제되었습니다."), redirect(admin.index)
    end
```

관리자가 동영상 삭제 버튼을 눌렀을 때의 흐름입니다.

sequenceDiagram

## 6. 댓글 삭제 시퀀스

관리자가 댓글 삭제 버튼을 눌렀을 때의 흐름입니다. `next=index`이면 통합 관리 페이지로, 없으면 댓글 목록 페이지로 돌아갑니다.

```mermaid
sequenceDiagram
    participant U as 관리자
    participant F as admin.comment_delete
    participant DB as DB

    U->>F: POST /admin/comments/<comment_id>/delete (csrf_token, next?)
    F->>DB: Comment.query.get_or_404(comment_id)
    F->>DB: db.session.delete(comment), db.session.commit()
    DB->>F: OK

    alt next == "index"
        F->>U: redirect(admin.index)
    else 그 외
        F->>U: redirect(admin.comments, page, q)
    end
    F->>U: flash("댓글이 삭제되었습니다.")
```

## 7. 관리자 권한 판단 요약

```mermaid
flowchart LR

```

## 8. 관련 라우트·역할

| 경로 | 메서드 | 설명 |
|------|--------|------|
| `/admin` | GET | 관리자 통합 대시보드 (통계 + 사용자/동영상/댓글 표) |
| `/admin/users` | GET | 회원 목록 (검색·페이지네이션) |
| `/admin/users/<id>/edit` | GET, POST | 회원 수정 (이메일, 닉네임, 비밀번호, 프로필, 관리자 여부) |
| `/admin/users/<id>/delete` | POST | 회원 삭제 (자기 자신 제외) |
| `/admin/videos` | GET | 동영상 목록 |
| `/admin/videos/<id>/delete` | POST | 동영상 삭제 |
| `/admin/comments` | GET | 댓글 목록 |
| `/admin/comments/<id>/delete` | POST | 댓글 삭제 |

모든 `/admin/*` 라우트는 `@login_required` + `@_admin_required` 로 보호됩니다.
